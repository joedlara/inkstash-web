## Task 7: `create-drop-payment-intent` edge fn

`supabase/functions/create-drop-payment-intent/index.ts`. Steps:
1. Auth check
2. Load drop. Reject 404 if missing.
3. Call `reserve_drop_capacity(p_drop_id)`:
   - throws `not_yet_live` → return 400 `{ error: 'not_yet_live' }`
   - returns false → return 400 `{ error: 'sold_out' }`
   - returns true → continue
4. Build PaymentIntent per drop.kind:
   - `listing`: load listing, create destination-charge PI to seller's Connect with `application_fee_amount`, metadata.payment_type='listing' + drop_id
   - `pack`: build a vendor_pack-style PI (reuse the create-payment-intent logic by calling it directly), metadata.payment_type='vendor_pack' + drop_id
5. Return `{ client_secret, drop_id }`

Deploy. Commit.

## Task 8: Webhook side-effect — record drop_id on orders

**Decision: fold orders.drop_id into Task 1's migration so the webhook write is non-error.**

Deploy webhook. Commit.
